排序查询 与 自定义排序

排序查询：

注意：多个字段排序，先按第一个字段和方式进行排序，结果相同再按第二个字段和方式排序….

排序方式：

升序（小到大）：ASC，可以不写，默认为升序

降序（大到小）：DESC

自定义排序：

MySQL 会查看 字段名 的值。如果值等于 值1，排序权重为 1。如果值等于 值2，排序权重为 2。以此类推。

注意：null值不参与所有聚合函数的运算。

分组查询

where 与 having 的区别：

1.执行时间不同：where 是分组之前进行筛选，不满足 where 条件，不参与分组；having 是分组之后对分组结果进行过滤。

2.判断条件不同：where 不能对聚合函数进行判断，二 having 可以。

注意：分组之后，查询的字段一般为分组字段和聚合函数。

分页查询

limit（MySQL方言）

说明：

1.索引从0开始。

2.分页查询是数据库的方言，不同数据库有不同的实现，MySQL 中是 LIMIT。

3.SQL 语句中，如果起始索引为0，则起始索引可以省略，直接简写为: limit 查询记录数。

4.起始索引计算公式：（页码 - 1）* 每页记录数

例：每页5条数据，要返回第七页的五条数据就是

limit 30, 5

在实际表数据连接的过程中，经常需要在连接时跟上一个限制的连接条件，满足条件的数据行才能进行连接，这就是有条件连接。

自关联查询

两个表进行关联时，如果左表和右边是同一张表，这就是自关联。

左右表是同一张表，自关联时，表必须起别名

使用场景：

1.处理具有层级关系，比如省市区、员工与经理等

2.比较同一张表中不同行数据，比如计算每个月的销售额和上个月销售额的差值。（表a.month - 表b.month = 1）

子查询

在一个 SELECT 语句中，嵌入了另外一个 SELECT 语句，那么被嵌入的 SELECT 语句称之为子查询语句，外部那个SELECT 语句则称为主查询。

作用：子查询是辅助主查询的。

子查询的结果充当主查询的条件
子查询的结果充当主查询的数据源(临时表)
子查询的结果充当主查询的查询字段
